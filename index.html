Optimized MapFile based Storage of Small Files

The Hadoop Distributed File System (HDFS)is a distributed file system designed to run on commodity hardware and is suitable for applications that have large data sets. HDFS does not perform well for large number of small files since they put a lot of burden on NameNode of HDFS both in terms of memory and access time. To overcome these defects, we merge small files into larger file and store the merged file on HDFS. Generally, when small files are merged, variation in size distribution of files is not taken into consideration. We propose a new algorithm OMSS (Optimized MapFile based Storage of Small files) which merges the small files into larger files based on Worst fit strategy that helps in reducing internal fragmentation in data blocks, which in turn leads to fewer data blocks consumed for same number of small files
